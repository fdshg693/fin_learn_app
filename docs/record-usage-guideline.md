# record 利用ガイドライン

更新日: 2026-02-15

## 方針
`record` は「値として扱う」ことが本質の型に限定して使う。

## 使う
- Value Object（例: `Money`, `CompanyId` など）
- Command / DTO のような immutable なデータ運搬型（必要に応じて）
- Domain Event（イベント内容を値として持つ）

## 使わない
- 振る舞い中心で、値比較を前提にしない型
- サービス結果で等価比較が不要な型

## このプロジェクトでの適用
- `ActionExecutionResult` は `record` から `class` へ変更済み
  - 理由: 結果オブジェクトとしての等価比較が不要

## 判断チェックリスト
1. この型は「同じ値なら同じもの」と見なしたいか？
2. `with` 式や値比較が将来必要か？
3. 変更よりも不変なデータ運搬が主目的か？

上記が YES なら `record`、そうでなければ `class` を優先する。
